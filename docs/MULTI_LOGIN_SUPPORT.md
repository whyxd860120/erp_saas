# 多种登录方式支持

## 功能说明

现在系统支持使用**用户名、姓名或邮箱**进行登录，提供更灵活的登录体验。

## 支持的登录方式

### 1. 邮箱登录
使用注册时的邮箱地址登录。
```
示例：admin@erp2026.com
```

### 2. 姓名登录
使用用户姓名登录。
```
示例：张三、管理员、System Admin
```

### 3. 用户名登录
如果系统支持用户名字段，也可使用用户名登录。

## 实现细节

### 前端修改

#### 1. 登录表单
- 将输入框提示从"请输入邮箱"改为"用户名 / 姓名 / 邮箱"
- 修改表单验证规则，移除邮箱格式验证

#### 2. API 参数
- 修改 `LoginParams` 接口，将 `email` 改为 `username`
- 保持向后兼容，后端仍可处理多种输入格式

### 后端修改

#### 1. 系统登录 (`systemLogin`)
```typescript
// 支持三种方式查询用户
user = await prisma.systemUser.findFirst({
  where: {
    OR: [
      { email: username },
      { name: username },
    ],
  },
});
```

#### 2. 租户登录 (`tenantLogin`)
```typescript
// 支持三种方式查询用户
const user = await prisma.user.findFirst({
  where: {
    OR: [
      { email: username },
      { name: username },
    ],
    ...(tenantId && { tenantId }),
  },
  include: { tenant: true },
});
```

### 数据库优化

#### 1. 索引优化
为提高查询性能，在数据库中添加了索引：

```prisma
// 系统用户表
model SystemUser {
  ...
  @@index([email])
  @@index([name])  // 新增
  @@map("system_users")
}

// 租户用户表
model User {
  ...
  @@index([email])
  @@index([name])  // 新增
  @@map("users")
}
```

## 优势

1. **用户体验提升**
   - 用户不需要记住邮箱，可以使用更方便的姓名登录
   - 支持多种登录方式，减少输入错误

2. **灵活性增强**
   - 不同场景下可以使用不同的登录方式
   - 临时访问人员可使用姓名快速登录

3. **向后兼容**
   - 原有的邮箱登录方式仍然支持
   - 不影响现有用户的登录习惯

## 注意事项

1. **唯一性验证**
   - 邮箱在租户内唯一（User表）
   - 邮箱在系统内唯一（SystemUser表）
   - 姓名可能不唯一，但查询时会返回第一个匹配的用户

2. **优先级**
   - 系统优先匹配邮箱，然后匹配姓名
   - 如果出现重复姓名，建议使用邮箱登录

3. **性能优化**
   - 已为 `email` 和 `name` 字段添加索引
   - 使用 Prisma 的 `findFirst` 查询优化性能
