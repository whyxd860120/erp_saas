# 菜单管理功能使用指南

## 📍 访问路径

在浏览器中访问 `/menus` 即可进入菜单管理页面（需要管理员权限）。

## 🎯 主要功能

### 1. 查看菜单结构
- 树形结构展示所有菜单
- 显示菜单层级（L1=一级，L2=二级，L3=三级）
- 显示菜单编码、路由路径、图标等信息
- 系统菜单有特殊标识（橙色背景+系统标签）

### 2. 拖拽调整顺序和层级

#### 调整顺序
- 拖拽菜单项到同级菜单的上方或下方
- 可以改变菜单的排列顺序

#### 调整层级
- **设为子菜单**：拖拽到某个菜单上（当该菜单不是系统菜单时）
- **设为一级别**：通过菜单右侧的"操作 → 设为一级菜单"

#### 层级限制
- 最多支持3级菜单（L1 → L2 → L3）
- 系统菜单不能作为拖拽目标（防止误操作）

### 3. 新增菜单

#### 新增一级菜单
- 点击右上角的"新增菜单"按钮

#### 新增子菜单
- 点击菜单右侧的"操作 → 新增子菜单"
- 或在编辑菜单时选择父级菜单

### 4. 编辑菜单
- 点击菜单右侧的"操作 → 编辑"
- 可以修改菜单名称、编码、路由路径、图标、排序等
- 系统菜单的编码不能修改

### 5. 删除菜单
- 点击菜单右侧的"操作 → 删除"
- 如果菜单有子菜单，需要先删除子菜单
- 系统菜单不能删除

### 6. 保存修改
- 拖拽操作完成后，点击右上角的"保存修改"按钮
- 所有调整才会生效

## 🎨 操作提示

页面右侧有操作提示面板，帮助你了解如何操作：

1. **拖拽到同级菜单上方/下方** → 调整顺序
2. **拖拽到某个菜单上** → 设置为子菜单
3. **通过"操作→设为一级菜单"** → 提升层级
4. **点击"保存修改"才能生效** → 重要提示

## 📋 菜单属性说明

| 属性 | 说明 | 示例 |
|------|------|------|
| 菜单名称 | 显示名称 | 供应商管理 |
| 菜单编码 | 唯一标识 | supplier |
| 路由路径 | URL路径 | /suppliers |
| 图标 | Element Plus图标名 | OfficeBuilding |
| 父级菜单 | 上级菜单 | 基础资料 |
| 排序 | 数字越小越靠前 | 1 |
| 描述 | 备注信息 | 管理供应商信息 |

## ⚠️ 注意事项

1. **系统菜单**：标记为"系统"的菜单是内置菜单，不能删除，编码不能修改
2. **层级限制**：最多支持3级菜单
3. **保存生效**：拖拽操作需要点击"保存修改"才能生效
4. **子菜单**：删除有子菜单的菜单前需先删除子菜单

## 🔧 技术实现

### 前端
- 组件：`frontend/src/views/MenuListView.vue`
- API：`frontend/src/api/menu.ts`

### 后端
- Controller：`src/controllers/menu.controller.ts`
- 路由：`src/routes/menu.routes.ts`

### 数据库
- 表：`permissions`（type = 'menu'）
- 关键字段：`parentId`（父菜单ID）
