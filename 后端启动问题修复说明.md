# 后端启动问题修复说明

## 🐛 问题描述

**错误信息**:
```
error TS5095: Option 'bundler' can only be used when 'module' is set to 'preserve' or to 'es2015' or later.
error TS5103: Invalid value for '--ignoreDeprecations'.
[nodemon] app crashed - waiting for file changes before starting...
```

**根本原因**:
`ts-node` 版本与 TypeScript 5.3.3 存在兼容性问题，导致编译失败。

---

## ✅ 已完成的修复

### 1. 替换 ts-node 为 tsx
- **修改文件**: `package.json`
- **变更内容**: 
  ```json
  // 修改前
  "dev": "nodemon --exec ts-node --transpile-only src/index.ts"
  
  // 修改后
  "dev": "nodemon --exec tsx src/index.ts"
  ```

### 2. 安装 tsx
```bash
npm install tsx --save-dev
```

### 3. 清理并重启
- 清理了旧的 `dist` 目录
- 重新生成 Prisma 客户端
- 在新窗口中启动服务

## 🔍 如果仍有问题

### 情况 1: 新窗口显示相同错误

**解决方法**:
1. 关闭所有 Node.js 进程
   ```powershell
   taskkill /F /IM node.exe
   ```

2. 删除 node_modules 并重新安装
   ```bash
   rm -rf node_modules package-lock.json
   npm install
   ```

3. 重新启动
   ```bash
   npm run dev
   ```

### 情况 2: 端口 3000 被占用

**解决方法**:
1. 查找占用进程
   ```powershell
   netstat -ano | findstr ":3000"
   ```

2. 终止进程
   ```powershell
   taskkill /F /PID <进程ID>
   ```

3. 重新启动服务

### 情况 3: 数据库连接失败

**检查项**:
1. MySQL 服务是否运行
   ```powershell
   Get-Service MySQL
   ```

2. `.env` 配置是否正确
   ```env
   DATABASE_URL="mysql://root:Esoft123@456@localhost:3306/erpnext_db?charset=utf8mb4"
   ```

## 💡 为什么使用 tsx？

| 特性 | ts-node | tsx |
|------|---------|-----|
| 速度 | 较慢 |  更快（使用 esbuild） |
| 兼容性 | 有时与 TS 5.x 冲突 | ✅ 更稳定 |
| 维护 | 较少更新 | ✅ 积极维护 |
| 配置 | 需要复杂配置 | ✅ 零配置 |

**tsx** 是一个更快的 TypeScript 执行器，基于 esbuild，对 TypeScript 新版本有更好的支持。
