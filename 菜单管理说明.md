# 菜单管理功能说明

## 🔍 问题诊断

如果菜单管理页面没有显示任何菜单，请按以下步骤操作：

### 第一步：运行诊断脚本

双击运行 **`诊断菜单数据.bat`**，查看数据库中是否有菜单数据。

### 第二步：根据诊断结果处理

#### 情况1：数据库中没有菜单数据

**原因**：权限数据未初始化

**解决方案**：
- **方案A**：重启后端服务器
  - 后端启动时会自动执行 `initPermissions()` 函数
  - 会根据 `defaultPermissions` 数组初始化所有权限（包括菜单和按钮）

- **方案B**：运行初始化脚本
  - 双击运行 **`初始化菜单数据.bat`**
  - 这个脚本会创建或更新所有菜单数据

#### 情况2：数据库中有菜单数据但页面不显示

**检查项**：
1. ✅ 后端服务器是否正常运行？
2. ✅ 前端开发服务器是否运行？
3. ✅ 浏览器控制台是否有错误信息？
4. ✅ API请求是否成功？

**解决方法**：
```bash
# 重启后端
npm run dev

# 重启前端
cd frontend
npm run dev
```

## 📊 菜单数据初始化流程

### 自动初始化（推荐）

**后端启动时自动初始化**：

1. 后端服务器启动
2. 执行 `app.ts` 中的 `initPermissions()`
3. 检查数据库是否已有权限数据
4. 如果没有，创建所有权限（菜单+按钮）
5. 如果有，跳过初始化

**代码位置**：
- 后端入口：`src/app.ts` 第 239 行
- 初始化函数：`src/controllers/permission.controller.ts` 第 122 行
- 权限数据定义：`src/controllers/permission.controller.ts` 第 10 行

### 手动初始化

**使用脚本初始化**：

1. 双击运行 **`初始化菜单数据.bat`**
2. 脚本会自动：
   - 连接到数据库
   - 查询现有菜单
   - 创建缺失的菜单
   - 更新已存在的菜单
   - 显示最终菜单结构

## ⚠️ 注意事项

1. **权限类型**：数据库中权限有三种类型：
   - `menu` - 菜单权限
   - `button` - 按钮权限
   - `api` - API权限

2. **父子关系**：通过 `parentId` 字段建立菜单层级关系

3. **系统权限**：标记为 `isSystem: true` 的权限是系统内置，不能删除

4. **编码唯一**：每个权限的 `code` 必须唯一

## 🆘 常见问题

### Q: 运行诊断脚本报错 "未找到 node_modules"
**A**: 需要先安装依赖，运行 `npm install`

### Q: 初始化脚本连接数据库失败
**A**:
1. 检查 `.env` 文件中的数据库配置
2. 确保数据库服务正在运行
3. 检查数据库连接信息是否正确

### Q: 菜单显示但没有样式
**A**: 重启前端开发服务器，让最新的代码生效

### Q: 拖拽不生效
**A**: 确保在拖拽后点击"保存修改"按钮
