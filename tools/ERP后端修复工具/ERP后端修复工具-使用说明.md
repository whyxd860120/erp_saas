# 🛠️ ERP 后端修复工具 - 使用指南

## 📦 工具介绍

**ERP后端修复工具.hta** 是一个中文图形化界面的自动化修复工具，专门用于解决 ERP 系统后端服务的常见问题。

### ✨ 主要特点

- 🎨 **中文界面** - 完全中文化，操作简单直观
- 🔍 **自动诊断** - 一键检测所有环境问题
- 🔧 **智能修复** - 自动修复配置错误
- 🚀 **一键启动** - 自动启动后端服务
- 📊 **实时反馈** - 详细的日志和状态显示
- ✅ **自我验证** - 已测试通过，可安全使用

---

## 🚀 快速开始（3步搞定）

### 步骤 1: 启动工具
双击运行：
```
ERP后端修复工具.hta
```

工具会自动打开一个漂亮的图形界面。

### 步骤 2: 点击"全面检查"
在工具界面中，点击 **"🔍 全面检查"** 按钮。

工具会自动检测：
- ✅ 配置文件是否存在
- ✅ MySQL 服务是否运行
- ✅ Node.js 是否安装
- ✅ 后端服务是否正常

### 步骤 3: 点击"一键修复"
如果检查发现问题，点击 **"🔧 一键修复"** 按钮。

工具会自动：
1. 备份当前配置
2. 修复数据库密码不一致问题
3. 验证 MySQL 连接
4. 启动后端服务
5. 测试 API 接口

## 🔍 常见问题

### Q1: 提示"MySQL 连接失败"？
**解决方法**:
1. 检查 MySQL 服务是否运行
   ```powershell
   Get-Service MySQL
   ```
2. 如果未运行，启动它：
   ```powershell
   Start-Service MySQL
   ```
3. 重新点击"全面检查"

### Q2: 提示"Node.js 未找到"？
**解决方法**:
1. 安装 Node.js (>= 18.0.0)
   - 下载: https://nodejs.org/
   - 选择 LTS 版本
2. 安装完成后重启工具

### Q3: 后端服务启动失败？
**解决方法**:
1. 查看新打开的后端服务窗口
2. 阅读窗口中的错误信息
3. 常见原因：
   - 端口 3000 被占用
   - 依赖未安装（运行 `npm install`）
   - 数据库连接失败

### Q4: 修复后仍然 502 错误？
**解决方法**:
1. 等待 15-30 秒，让服务完全启动
2. 点击"🏥 健康检查"再次验证
3. 查看后端服务窗口的日志
4. 刷新浏览器页面

### Q5: 如何查看后端服务日志？
**答**: 
- 后端服务会在新窗口中运行
- 该窗口显示所有实时日志
- 保持该窗口打开，关闭它会停止服务

## 💡 使用技巧

### 技巧 1: 定期健康检查
建议每天点击"🏥 健康检查"一次，确保系统正常运行。

### 技巧 2: 查看日志
遇到问题时，先查看日志窗口的详细信息，通常能找到问题原因。

### 技巧 3: 备份文件
每次修复都会自动备份 `.env` 为 `.env.backup`，如有问题可以恢复。

### 技巧 4: 保持服务窗口打开
后端服务在新窗口中运行，**不要关闭该窗口**，否则服务会停止。
